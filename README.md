# 🎮 Steam 游戏数据分析：Tag 机会发现与立项辅助系统 (V2)

**🔗 在线体验地址**: [Steam 决策辅助系统 (V2)](https://steam-data-analysis-v2-lv8eu5t3vi98rr2mjnk2ck.streamlit.app/)

本项目从最初的“数据可视化分析工具”升级为一个**以业务决策为导向的辅助系统**，专为游戏策划、产品经理和投资/研运人员设计。通过深度挖掘 Steam 全量游戏数据，帮助团队在**预研、立项和竞品分析阶段**，科学地验证“品类方向”与“Tag 组合”的市场潜力。

---

## 1. 为什么要做这个项目？ (Motivation)
目前市面上关于 Steam 游戏的数据平台（如 SteamDB、SteamSpy 等）虽然信息丰富，但在实际的“游戏立项与商业预研”场景中，仍存在明显的痛点：
1. **纯检索导向，缺乏分析结论**：现有平台多为“黄页型”词典，只能查询单一游戏的数据，无法直接回答“A+B 两个玩法 Tag 组合在一起，是否能产生 1+1>2 的商业增益？”等核心决策问题。
2. **底层数据集缺陷**：Kaggle 等开源数据集通常样本量过小（例如只有畅销榜 Top 1000），无法真实反映长尾市场的全貌，极易导致“幸存者偏差”与分析过拟合；同时，官方 API 本身**不直接提供玩家自定义的 Tags 列表**，导致很难做深度的品类交叉分析。

为了从数据视角复刻“**如何发现值得做成表层卖点的题材/玩法**”这一过程，我们决定从底层爬虫做起，构建一套完全自主、可复用、且覆盖 Steam 全量活跃游戏的数据与决策平台。

## 2. 数据工程架构：如何搞定 ~16.5 万款游戏的数据底座？
为了打破数据源的限制，我们摒弃了“现成小样本 CSV”，设计了一套高可用的**混合式自动化数据抓取与入库流水线 (Pipeline)**，成功对 Steam 全库近 16.5 万个 AppID 进行了全量摸排，并最终筛选出约 1.9 万款具有统计价值的有效商业游戏（评论数 > 200）。

### 🛠️ 混合获取方案 (Hybrid Data Fetching)
*   **基础数据 (API 高速拉取)**：采用官方 `appreviews` 接口，无头静默拉取每个游戏的总评论数（Review Count）与好评率（Positive Rate）。
*   **深层数据 (Store 页面爬虫)**：由于官方 API 不返回玩家自定义 Tag，我们针对通过初筛（热度达标）的游戏，直接向 Steam 商店 HTML 页面发起请求，通过 CSS Selector 精准刮取发行日期与前 5 个核心 Tag。

### ⚡ 并发提效与高可用设计
面对海量的 HTTP 请求与 Steam 严格的反爬虫限制（IP 限流），我们设计了如下策略：
*   **漏斗式初筛**：将官方 API 作为第一道廉价漏斗，只有评论数 > 200 的有效游戏才会被放入“待深度爬取”的队列，极大地节省了网页请求的开销。
*   **SQLite 状态持久化 (Checkpoint)**：引入本地 SQLite 数据库作为缓冲队列。每个游戏的爬取状态都会记录 `last_updated`，实现断点续传。无论断网还是关机，重启后脚本都会自动从上一个成功点继续，不浪费一秒钟。
*   **多分片并行执行 (Hash-based Sharding)**：为了打破单线程瓶颈，我们将全量 AppID 通过稳定的 `hash(appid) % 5` 算法切分为 5 个完全互斥的子数据库（Shard）。通过开启 5 个并发后台守护进程（Daemon Worker）同时抓取，在安全请求频率下，将原本需要一星期的爬取时间压缩到了十几个小时。

最终产生的数据集被清洗并合并为一份高价值基准库，导入至云端 **Supabase (PostgreSQL)**，供前端秒级查询。该爬虫 Workflow 代码已完全解耦并模块化，未来每半年运行一次即可完成全量自动化更新。

## 3. 前端决策引擎设计 (V2 架构)
基于 Streamlit 搭建的前端，我们在 V2 版本中完成了从“底层数据看图”到“宏观业务看表”的体验重构，划分为**两大决策场景**：

### 🌟 模块一：机会发现 (Opportunity Discovery)
**回答问题：“当前有哪些值得优先研究的方向？”**
*   **宏观榜单直给**：不再让用户在散点图里大海捞针，直接通过统计算法给出“高口碑低供给”、“高口碑高热度”、“近3年升温最快”及“高热低口碑预警（避坑排雷）”的 Top 10 榜单。
*   **机会地图**：引入对数坐标系（Log-scale）的供给/质量散点图，直观展现各细分品类的红海与蓝海分布。

### 🎯 模块二：方向研究 (Direction Validation)
**回答问题：“我想做的这个方向/组合，到底靠谱吗？”**
*   **多 Tag 方向验证（核心引擎）**：
    *   **五档评价体系**：自研 Verdict 算法。用户输入 2~3 个拟融合的 Tag（如 Action + Roguelike），系统自动输出客观的 **5 维体检结果（口碑、热度、协同增益、动量趋势、置信度）**。
    *   **协同增益 (Synergy) 算法**：突破性地引入 `Lift` 概念。计算 `P(高好评 | TagA+TagB) / P(高好评 | 单一Tag)`。不仅告诉你两个 Tag 加起来厉不厉害，更告诉你它们是不是产生了“1+1>2”的化学反应。
*   **单 Tag 画像**：
    *   **赛道全景**：展示单赛道热度中位数、好评率中位数、时间趋势。
    *   **协同关系网**：算出与该 Tag 组合后“增益最高”与“最易翻车（被稀释）”的搭配套路。
*   **案例库 / 对标寻找**：
    *   输入任意一款对标游戏，系统自动进行 Tag 基因匹配，不仅展示该游戏的绝对数据，还自动拉出“与它拥有相似 Tag 基因，但在市场上表现**更好**和**更差**”的对标产品，帮助策划寻找“微创新空间”。
